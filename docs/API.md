# AI Culler — Scripting API

All four functions are registered under the `AIC` tag. Call them from any script running on the server or a client — the server-mutating calls auto-forward to the server via `remoteExec` if invoked from a client machine.

---

## AIC_fnc_protect

**Marks a unit as protected — permanently excluded from culling.**

```sqf
[_unit] call AIC_fnc_protect;
```

| Parameter | Type | Description |
|---|---|---|
| `_unit` | Object | The AI unit to protect |

**Notes**
- If called from a client, the call is forwarded to the server via `remoteExec` automatically.
- If the unit is currently culled, it is re-enabled immediately.
- Players and vehicles are silently ignored.
- The flag is broadcast globally (`setVariable ["AIC_zeusProtected", true, true]`).

**Example**
```sqf
// Protect a specific unit (server or client)
[myBossUnit] call AIC_fnc_protect;
```

---

## AIC_fnc_unprotect

**Removes protection from a unit, returning it to the normal culling pool.**

```sqf
[_unit] call AIC_fnc_unprotect;
```

| Parameter | Type | Description |
|---|---|---|
| `_unit` | Object | The AI unit to unprotect |

**Notes**
- If called from a client, the call is forwarded to the server via `remoteExec` automatically.
- The unit will be evaluated normally on the next culler tick.
- Players and vehicles are silently ignored.

**Example**
```sqf
// Unprotect after a scripted event ends
[myBossUnit] call AIC_fnc_unprotect;
```

---

## AIC_fnc_isCulled

**Returns whether a unit is currently culled (simulation disabled).**

```sqf
private _culled = [_unit] call AIC_fnc_isCulled;
```

| Parameter | Type | Description |
|---|---|---|
| `_unit` | Object | The unit to check |

**Returns** `Boolean` — `true` if the unit's simulation is currently disabled by the culler.

**Notes**
- Safe to call from any machine (reads a locally-synced variable).
- Returns `false` for null objects.

**Example**
```sqf
if ([myUnit] call AIC_fnc_isCulled) then {
    hint "That unit is currently culled.";
};
```

## AIC_fnc_getStats

**Returns a snapshot of the last culler tick's statistics.**

```sqf
private _stats = [] call AIC_fnc_getStats;
```

**Returns** `HashMap` with the following keys:

| Key | Type | Description |
|---|---|---|
| `"active"` | Number | Units currently simulating |
| `"los"` | Number | Units active because a player has LOS |
| `"noLos"` | Number | Units active due to proximity but no LOS |
| `"culled"` | Number | Units with simulation disabled |
| `"protected"` | Number | Units excluded from culling |
| `"override"` | Number | Units active due to a Zeus waypoint |
| `"total"` | Number | All living managed AI (active + culled + protected) |
| `"serverFps"` | Number | Server FPS at the last tick |

**Notes**
- Returns an empty HashMap before the first culler tick completes.
- Updated server-side on every culler tick — read from any machine once `AIC_lastStats` has been populated.
- On clients, `AIC_lastStats` is not broadcast; call this function from the server or read from Zeus-side code that has access to the server namespace.

**Example**
```sqf
private _stats = [] call AIC_fnc_getStats;
private _active = _stats getOrDefault ["active", 0];
private _total  = _stats getOrDefault ["total",  0];
systemChat format ["Active: %1 / %2", _active, _total];
```

## Compatibility with Third-Party Mods

If your mod places AI and assigns waypoints via script, those waypoints are treated the same as Zeus-placed waypoints — the group will be activated (Override state) within one poll interval (default 5s). No additional configuration is needed.

If you need a specific unit to always simulate regardless of player distance or the active cap, use `AIC_fnc_protect` rather than trying to manipulate `AIC_zeusProtected` directly — the API function handles the enable call and label update in one step.

### LAMBS / Vcom / Reinforcement Mods

Any mod that creates units via script and assigns waypoints will have those groups detected by the waypoint monitor. The waypoint monitor only fires during non-combat periods, so combat waypoints generated by LAMBS/Vcom (MOVE/SAD orders in response to contact) will not incorrectly set Override state.

If a third-party mod needs to temporarily prevent a unit from being culled during a scripted sequence, wrap the sequence with `protect` and `unprotect`:

```sqf
// Before scripted sequence
[myUnit] call AIC_fnc_protect;

// ... scripted events ...

// After scripted sequence
[myUnit] call AIC_fnc_unprotect;
```
