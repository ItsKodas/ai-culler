[h1]AI Culler: Optimized Warfare[/h1]
A two-part performance mod for Arma 3 that tackles both the server-side simulation overhead and the client-side render cost of large AI counts. Built for milsim ops running 60+ players against 150+ AI.

[h1]The Problem[/h1]
[b]Server:[/b] Arma 3 simulates every AI unit simultaneously on a fixed cycle. Beyond ~150 units this causes periodic frame spikes even with LAMBS and Headless Clients.

[b]Client:[/b] Arma 3's renderer has no robust occlusion culling — looking toward a large AI force drops FPS even when every unit is behind a building or hill.

[h1]The Solution[/h1]
[b]aic_main[/b] (server + Zeus) caps the number of actively-simulating AI and prioritises units by proximity and line of sight. Vehicles are never culled. Protected, in-combat, and Zeus-ordered units are always active regardless of distance.

[b]aic_client[/b] (every player client) hides occluded AI models via 'hideObject', removing them from each player's render pipeline. Trees and bushes don't count as occlusion. Units within 75m are always rendered to prevent pop-in. An ADS-cone bypass keeps units visible while you're aiming at them. The occlusion radius is automatically read from each player's view distance (ACE3 or native) and kept in sync throughout the mission.

Both addons ship in one mod — the right code runs on the right machine automatically.

[h1]Live Zeus Control[/h1]
A status window opens automatically when Zeus is active. Adjust max active AI, per-faction cull distances, check interval, and combat detection radius on the fly — changes broadcast to the server instantly. A live FPS sparkline tracks server performance over the last 88 seconds with current, average, min, and max in the title bar. No restart, no rebuild.

[h1]CBA Addon Options[/h1]
All settings are exposed under [b]Configure → Addon Options[/b] and persist between sessions. Server settings (cull distances, AI cap, intervals) are server-enforced and broadcast to all clients. Client settings (label display, safe radius, enable toggle) are per-player. Requires CBA_A3; falls back to hard-coded defaults without it.

[h1]Features[/h1]
[list]
[*] Per-faction cull distances (BLUFOR / OPFOR / Independent / Civilian)
[*] AI-vs-AI combat auto-activation so distant firefights don't freeze
[*] Zeus waypoint override — assigned groups activate regardless of player distance
[*] Right-click [b]Toggle Culler Protection[/b] context action
[*] ZEN Enhanced Zeus context-menu integration
[*] 3D floating labels above managed units showing [b]Protected[/b], [b]Culled[/b], or [b]Override[/b] state
[*] Live server FPS sparkline graph in Zeus
[*] Client renderer debug HUD (server-controlled, activates on all clients simultaneously)
[*] No mission changes required — no 'init.sqf' entry
[/list]

[h1]Installation[/h1]
[list]
[*] Load '@ai_culler' on [b]the server, all Zeus clients, and every player client[/b]
[*] Add '-mod=@ai_culler' to launch parameters
[*] Defaults are safe for most ops — tune live from the Zeus panel or via Addon Options
[/list]

[h1]Compatibility[/h1]
[b]Tested:[/b] CBA_A3, LAMBS Danger, Headless Clients, Civilian Presence Module, Zeus / Curator, ACE3 (view distance auto-sync), ZEN Enhanced Zeus
[b]Untested:[/b] Vcom AI — may conflict
